> Last updated: 2026-06-16

## なぜ version-up するか

2月版（Phase 1/2）は強い理念だが、**主役が "Git / GitHub Enterprise" になっていて、自分のマニフェストと少し矛盾**している。MANIFESTO は「シャベルやジーンズを売るな」と言うのに、README は GitHub Enterprise というシャベルを売る構図に見える。

2026年中盤、企業がAIの"実行層"を外部委託（BPO等）に出し始めた流れが、答えを教えてくれた：

> **Git も、データ保守（MDM）も、外注できる substrate。外注できないのは、その上の「意味・構造・関係＝オントロジー」。**

## 中核ピボット：Git → オントロジー

| | 2月版（Phase 1/2） | 理念 v2（Phase 3） |
|---|---|---|
| 主役 | Git / GitHub Enterprise（基盤） | **所有されたオントロジー / 意味の層** |
| 道具 | GitHub が答え | **Git・AIは交換可能な substrate（一例）** |
| 診断 | データ基盤の分断 | **"所有されたオントロジーの不在"**（AIはファイルを読めても、何を意味し・どう繋がるかを知らない） |
| テーゼ | 知識をバージョン管理 | **substrate は外注できる。ontology は所有せよ。** |

→ これで **マニフェストの矛盾が解消**（道具でなく原理が主役）し、理念が**特定ツールに依存しない＝陳腐化しない**形になる。

## 3軸に拡張（2月版は個人OS＋組織OSの2軸）

| 軸 | 対象 | 内容 |
|---|---|---|
| **個人OS** | 実務者個人 | Planning Sheet、逆算思考、情報4分類、抽象化（既存） |
| **組織OS** | 経営・組織 | 所有されたオントロジー＝知識の意味と関係の層（主役に格上げ） |
| **関係OS（NEW）** | 顧客・パートナー | 所有したオントロジーを土台に、**顧客・パートナーとAIで共進化**する（メーカー側B2B共創、win-win） |

## 魂はそのまま（Human-First を維持・補強）

- MANIFESTO の "The Gold Rush Is Over / Human-First AI" は**核として維持**。
- 補強：今の言語化＝**「AIはツール→同僚→組織」のラダー**、**「足りないのは人材じゃなく WHY＝現場知」**を織り込む。
- 一貫メッセージ：道具の話で終わらせない。**人間の経験と判断（現場知）こそ、AIと組んだ時の最大の武器**。

## 具体的な改訂対象

- **MANIFESTO v2**：ラダー（ツール→同僚→組織）＋ WHY=現場知 を1段落追加。"don't sell the shovel" と Git推しの矛盾を解消する一文。
- **README v2**：3レイヤーを「オントロジー主役」に再構成／"39%ギャップ＝所有オントロジーの不在" に診断を鋭く／**関係OS（第3軸）追加**／GitHubは "a substrate, not the point" と中立化。
- **Phase 3 planning**：関係OS（顧客・パートナー共進化）の具体化。Planning Sheet を **v3.2 → 最新 v6.1** に更新（Eval / ループ設計 / 情報4分類 が既に入った版）。
- **サイト**：オントロジー主役・3軸・道具中立の構成に。
